# Memory Policy

## Purpose
Memory stores stable preferences, approved context, and important decisions.

## Rules
- Memory is lane-scoped by default.
- No automatic cross-lane memory reading.
- The user controls memory sharing between lanes.
- Store concise summaries, not raw dumps.
- Prefer stable facts over temporary chatter.

## What to Store
- stable preferences
- important decisions
- active project context
- recurring constraints
- approved standards

## What Not to Store
- secrets
- passwords
- tokens
- private keys
- unnecessary personal data
- temporary noise

## Update Pattern
Memory should be updated through concise summaries after meaningful work, not constantly.
